Mac中使用binlog2sql进行数据回滚

总所周知数据库的日志文件是保证数据库安全的必要手段，若没有日志文件或者备份等手段，稍不留神的误操作便会迫不得已而跑路。建议开启binlog并配置binlog2sql工具进行日志的记录以及必要时刻的回滚。

1. binlog2sql的下载

git clone https://github.com/danfengcao/binlog2sql.git
cd binlog2sql
pip install -r requirements.txt

2. Mac中Mysql配置文件

在MacBook中安装的mysql默认是没有配置文件的，下面介绍一下配置相关文件的步骤。

首先可以使用命令查看mysql指定my.cnf配置文件的路径优先级：

mysql --help|grep 'my.cnf'

可以看出/etc/my.cnf文件是mysql首先查找的文件，因此我们选择把配置文件放在该目录下（原文链接）

这里我直接给出配置好的my.conf配置文件：

链接: https://pan.baidu.com/s/1MRuFmKZStThePgI9JcLs-Q 密码: jabf

注意不要使用mac自带的文本编辑器打开配置文件，这样会改变文件的编码格式，而MySQL的配置文件需要时ASNI/ASCII格式的，所以可以使用vim进行编辑。

把配置文件放入/etc/并修改文件权限为664，不然MySQL读取文件的时候权限会出错:

cp ~/Desktop/my.cnf /etc/
cd /etc
sudo chmod 664 my.cnf

在改完这个配置的之后，重启MySQL服务器会启动不了。不要慌，我们只需要重启电脑就可以了。

4. 数据回滚

可以使用binlog2sql把日志文件转为sql形式的语句查看日志所记录的sql操作。

# 来到binlog2sql安装目录
$ cd /usr/local/sbin/binlog2sql

# 执行python代码（注意sudo）
$ sudo python binlog2sql/binlog2sql.py -h127.0.0.1 -P3306 -u root -p'123456' -d test_user -t upload_media --start-file='mysql-bin.000001'

执行结果：

DELETE FROM `test_user`.`upload_media` WHERE `upload_id`=1 AND `user_id`=1 AND `media_name`='媒体文件1' AND `upload_time`='2021-06-10 18:45:35' AND `media_url`='media1' AND `media_class`=1 AND `media_module_id`=1 LIMIT 1; #start 10916 end 11336 time 2021-06-10 18:49:20
DELETE FROM `test_user`.`upload_media` WHERE `upload_id`=2 AND `user_id`=2 AND `media_name`='媒体文件2' AND `upload_time`='2021-06-10 18:47:10' AND `media_url`='media2' AND `media_class`=2 AND `media_module_id`=2 LIMIT 1; #start 10916 end 11336 time 2021-06-10 18:49:20
DELETE FROM `test_user`.`upload_media` WHERE `upload_id`=3 AND `user_id`=3 AND `media_name`='媒体文件3' AND `upload_time`='2021-06-10 18:47:13' AND `media_url`='media3' AND `media_class`=3 AND `media_module_id`=3 LIMIT 1; #start 10916 end 11336 time 2021-06-10 18:49:20
DELETE FROM `test_user`.`upload_media` WHERE `upload_id`=4 AND `user_id`=4 AND `media_name`='媒体文件4' AND `upload_time`='2021-06-10 18:47:16' AND `media_url`='media4' AND `media_class`=4 AND `media_module_id`=4 LIMIT 1; #start 10916 end 11336 time 2021-06-10 18:49:20

根据输出的start和end信息完成相应数据的回滚，以上删除的四条数据start为10916，end为11336，首先检查回滚sql是否正确

$ sudo python binlog2sql/binlog2sql.py -h127.0.0.1 -P3306 -u root -p'123456' -d test_user -t upload_media --start-file='mysql-bin.000001' --start-pos=10916 --end-pos=11336 -B

执行结果：

INSERT INTO `test_user`.`upload_media`(`upload_id`, `user_id`, `media_name`, `upload_time`, `media_url`, `media_class`, `media_module_id`) VALUES (4, 4, '媒体文件4', '2021-06-10 18:47:16', 'media4', 4, 4); #start 10916 end 11336 time 2021-06-10 18:49:20
INSERT INTO `test_user`.`upload_media`(`upload_id`, `user_id`, `media_name`, `upload_time`, `media_url`, `media_class`, `media_module_id`) VALUES (3, 3, '媒体文件3', '2021-06-10 18:47:13', 'media3', 3, 3); #start 10916 end 11336 time 2021-06-10 18:49:20
INSERT INTO `test_user`.`upload_media`(`upload_id`, `user_id`, `media_name`, `upload_time`, `media_url`, `media_class`, `media_module_id`) VALUES (2, 2, '媒体文件2', '2021-06-10 18:47:10', 'media2', 2, 2); #start 10916 end 11336 time 2021-06-10 18:49:20
INSERT INTO `test_user`.`upload_media`(`upload_id`, `user_id`, `media_name`, `upload_time`, `media_url`, `media_class`, `media_module_id`) VALUES (1, 1, '媒体文件1', '2021-06-10 18:45:35', 'media1', 1, 1); #start 10916 end 11336 time 2021-06-10 18:49:20

如果sql语句没有错误则在上面执行语句的基础上加” | mysql -h127.0.0.1 -P3306 -uroot -p’123456’ “，执行回滚语句：

$ sudo python binlog2sql/binlog2sql.py -h127.0.0.1 -P3306 -u root -p'123456' -d test_user -t upload_media --start-file='mysql-bin.000001' --start-pos=10916 --end-pos=11336 -B | mysql -h127.0.0.1 -P3306 -uroot -p'123456'

查看数据库：

mysql> SELECT * FROM upload_media;
+-----------+---------+---------------+---------------------+-----------+-------------+-----------------+
| upload_id | user_id | media_name    | upload_time         | media_url | media_class | media_module_id |

|         2 |       2 | 媒体文件2     | 2021-06-10 18:47:10 | media2    |           2 |               2 |
